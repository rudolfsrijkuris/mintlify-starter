---
title: "GitHub Sources"
description: "Connect repositories to auto-generate social posts from commits, PRs, and releases."
---

Postezi listens to GitHub webhooks and transforms repository activity into draft social posts using an LLM. Posts land in your review queue — you approve, edit, and publish from the dashboard.

---

## How it works

<Steps>
  <Step title="Connect GitHub">
    Authorize Postezi via OAuth. Postezi gets read access to your repos and the ability to install webhooks.
  </Step>
  <Step title="Pick a repository">
    Select which repo to watch. Postezi installs a webhook on it.
  </Step>
  <Step title="Activity happens">
    You push a commit, merge a PR, or publish a release. GitHub sends the event to Postezi.
  </Step>
  <Step title="Posts are drafted">
    The LLM transforms the event into platform-native drafts (one per connected platform) and places them in your queue with `review_status: pending`.
  </Step>
  <Step title="You review and publish">
    Open the Queue tab in Posts, edit any drafts, click Approve to open in Compose, and publish.
  </Step>
</Steps>

### Supported events

| Event | What gets drafted |
|-------|-------------------|
| `push` | Commit summary, build-in-public update |
| `pull_request` (merged) | Feature announcement, what changed |
| `release` (published) | Launch post with tag name and release notes |

## OAuth authorization

`GET /v1/github/oauth/authorize`

Returns a URL to start the GitHub OAuth flow. Redirect the user there.

```bash
curl "https://api.postezi.com/v1/github/oauth/authorize" \
  -H "Authorization: Bearer $POSTEZI_KEY"
```

```json
{
  "data": {
    "url": "https://github.com/login/oauth/authorize?..."
  }
}
```

After authorization, GitHub redirects back to Postezi which stores the connection and redirects to the dashboard with `?github_authed=true&connection_id=...`.

## List your repos

`GET /v1/github/repos`

Returns repos accessible to the connected GitHub account, including whether each is already connected.

```bash
curl "https://api.postezi.com/v1/github/repos" \
  -H "Authorization: Bearer $POSTEZI_KEY"
```

```json
{
  "data": {
    "repos": [
      {
        "full_name": "yourname/myapp",
        "name": "myapp",
        "description": "My SaaS app",
        "updated_at": "2025-06-18T20:00:00Z",
        "already_connected": false
      }
    ]
  }
}
```

## Connect a repository

`POST /v1/github/repos/connect`

Installs a webhook on the repo and starts watching it.

```bash
curl -X POST https://api.postezi.com/v1/github/repos/connect \
  -H "Authorization: Bearer $POSTEZI_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "connection_id": "018f4a2b-...",
    "repo_full_name": "yourname/myapp"
  }'
```


## Disconnect a repository

`DELETE /v1/github/connections/{id}`

Removes the connection and uninstalls the webhook. Future events from that repo will be ignored.

```bash
curl -X DELETE "https://api.postezi.com/v1/github/connections/018f4a2b-..." \
  -H "Authorization: Bearer $POSTEZI_KEY"
```
