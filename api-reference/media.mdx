---
title: "Media"
description: "Upload images for use in posts."
---

Images must be uploaded to Postezi's media storage before use in a post. The upload returns a URL you pass as `media_urls` when publishing.

---

## Upload an image

`POST /v1/media`

Accepts `multipart/form-data` with a `file` field.

<CodeGroup>
```bash cURL
curl -X POST https://api.postezi.com/v1/media \
  -H "Authorization: Bearer $POSTEZI_KEY" \
  -F "file=@screenshot.png"
```

```json Response
{
  "data": {
    "id": "018f4a2b-...",
    "url": "https://r2.postezi.com/media/...",
    "mime_type": "image/png",
    "size": 245120,
    "original_name": "screenshot.png"
  }
}
```
</CodeGroup>

### Constraints

| Constraint | Value |
|------------|-------|
| Max file size | 1 MB |
| Supported formats | `jpeg`, `png`, `gif`, `webp` |
| Max per post | 4 images |

### Using uploaded media in a post

Pass the returned `url` in the `media_urls` array when publishing:

```json
{
  "content": "New dashboard screenshot",
  "platforms": ["bluesky"],
  "media_urls": ["https://r2.postezi.com/media/..."]
}
```

<Note>
  The MCP tool `upload_media` accepts a public URL and fetches the image server-side — useful for agents that can't send multipart form data.
</Note>
