## list_accounts

Returns all connected social media accounts. Call this first to confirm which platforms are available before publishing.

**No parameters.**

```json
// Example response
[
  { "id": "018f...", "platform": "bluesky", "username": "you.bsky.social", "connected_at": "..." },
  { "id": "018f...", "platform": "linkedin", "username": "Your Name", "connected_at": "..." }
]
```

## publish_post

Publishes a post immediately to one or more connected platforms.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `content` | string | ✓ | Post text. Bluesky max 300 chars, LinkedIn max 3,000 chars, Threads max 500 chars. |
| `platforms` | string[] | ✓ | Platforms to publish to. Valid: `bluesky`, `linkedin`, `threads`. Must be connected. |
| `media_urls` | string[] | — | R2 URLs from `upload_media`. Max 4 images. |

```json
// Example response
{
  "post_id": "018f...",
  "content": "Shipped v2.0 today.",
  "status": "published",
  "published_at": "2025-06-19T10:00:00Z",
  "results": [
    { "platform": "bluesky", "success": true, "url": "https://bsky.app/..." },
    { "platform": "linkedin", "success": true, "url": "https://linkedin.com/..." }
  ]
}
```

## schedule_post

Queues a post for publishing at a future time. The post is delivered automatically.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `content` | string | ✓ | Post text. |
| `platforms` | string[] | ✓ | Platforms to publish to. |
| `scheduled_at` | string | ✓ | RFC3339 UTC timestamp. Must be in the future. Example: `2025-06-20T09:00:00Z` |
| `media_urls` | string[] | — | R2 URLs from `upload_media`. |

```json
// Example response
{
  "id": "018f...",
  "status": "scheduled",
  "scheduled_at": "2025-06-20T09:00:00Z",
  "platforms": ["bluesky"],
  "task_id": "018f..."
}
```

## list_posts

Returns posts for the authenticated user, newest first.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `status` | string | — | Filter by status: `draft`, `scheduled`, `published`, `failed`. Omit for all. |
| `limit` | number | — | Max results. Default `20`, max `100`. |
| `offset` | number | — | Pagination offset. Default `0`. |

```json
// Example response
{
  "posts": [...],
  "total": 42,
  "limit": 20,
  "offset": 0
}
```

---

## cancel_post

Cancels a scheduled post and removes it from the queue. Only posts with `status: scheduled` can be cancelled.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `post_id` | string | ✓ | UUID of the scheduled post to cancel. |

The post is returned with `status: draft`.

---

## upload_media

Fetches an image from a public URL and uploads it to Postezi's storage. Returns a URL to use in `publish_post` or `schedule_post`.

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `url` | string | ✓ | Publicly accessible image URL. |

```json
// Example response
{
  "id": "018f...",
  "url": "https://r2.postezi.com/media/...",
  "mime_type": "image/png",
  "size": 245120
}
```

**Constraints:** Max 1 MB, supported formats: `jpeg`, `png`, `gif`, `webp`.

<Note>
  `upload_media` fetches the image server-side from the URL — the agent doesn't need to send binary data.
</Note>
